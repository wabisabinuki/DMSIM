# 詳細な注意点

エンジン挙動とメカニズムごとの落とし穴をまとめる。JSON の書き方そのものは [JSON_SPEC.md](JSON_SPEC.md)、カード実装中に得た個別の知見は [imp_tips.md](imp_tips.md)。

## イベントの使い分け

- **召喚 ≠ 出た**: `summon` は召喚の呼び出し、`enter_battle`（`BattleZoneEnterEvent`）が実際に出た時。「このクリーチャーが出た時」は `enter_battle`。置換で別ゾーンへ置かれた場合 `enter_battle` は発行されない。`reason == "summon"`（`entered_by_summon`）で召喚由来を判定できる。
- **`draw` イベントは無い**: ドロー誘発は `zone_change`（`deck` → `hand`）+ `reason: {"in": ["draw", "replacement_draw"]}` で書く。`reason` を絞らないと山札サーチも拾う。
- **`shield_break` は S・トリガー誘発に使わない**: S・トリガー / G・ストライクは `SHIELD -> HAND` の移動試行（シールドチェック）で確認される。`ShieldBreakEvent` はブレイクされたこと自体の通知。
- `tap` / `untap` イベントは攻撃・ブロック・`tap` 効果・ターン開始アンタップから発行される。`battle_won` はパワーが上回った時だけ（同値では発火しない）。`card_chosen` は相手がこのカードを選んだ時の誘発に使う。
- 進化元になっているカードは誘発判定で止まる（`is_evolution_source`）。購読自体は維持されるため、別領域誘発は壊れない。

## 解決・キューまわり

- **効果優先度**: ターンプレイヤーの S・トリガー → 非ターンプレイヤーの S・トリガー → ターンプレイヤーのその他 → 非ターンプレイヤーのその他。同グループ内は発生元プレイヤーが順序を選ぶ。
- **connector の意味**: `after`（その後）は前の効果が失敗してもスキップされても実行。`then`（そうしたら）は直前の効果を「試みた」場合のみ実行。任意効果を候補があるのにスキップしたら「試みていない」。置換などで結果的に不成立でも、実行を試みたなら `then` へ進む。
- **「実行したときだけ各ターン1度を消費」**: `once_per_turn_gate`（consume せず判定）→ 本処理 → `once_per_turn_mark`（ここで消費）を `then` で繋ぐ。
- **トリガー時 vs 解決時の条件**: `condition` / `active_if` はトリガー発火時に評価される。解決時の盤面で判定したい条件は効果側の `type: "if"` に書く。
- **スナップショット継続性**: 誘発の解決前に発生源の `is_same_card()` が確認され、領域を離れていれば解決しない。「破壊された時、このカードを〜」のような自己参照誘発は `ignore_source_continuity: true` が必要（破壊移動で `zone_change_counter` が進むため）。
- **`first_time_each_turn` は `active_if` と独立に消費される**: 城が非アクティブな間に最初の攻撃が起きてもカウンターは消費される（DM ルール通り）。`active_if` で短絡しない。
- **保留状態**: 使用中・移動中のカードは他の効果から見えない。唱えている呪文自身は discard 対象にならない。マナから使うカード自身はマナ支払いに使える（唯一の例外）。保留解除時に保留前の一時効果・パワー修正は失われる。

## シールド関連

- シールドは**スロット**単位で数える（`visible_shields()` / `shield_count()`）。重なったカードは移動・ブレイクともスロット全体が動く。`shield_zone.cards` は物理カード枚数なので混同しない。
- シールドチェック（S・トリガー / G・ストライク）は最終移動先が手札の場合のみ宣言できる。置換で墓地等へ向かう場合は宣言不可。`s_trigger_suppressed` は S・トリガーの宣言可否のみに影響し、G・ストライクには影響しない。
- G・ストライクは公開時点では対象を選ばず、キュー解決時に相手クリーチャーを 1 体選んで攻撃禁止（ターン終了まで）。複数持ちは複数回誘発する。

### 城・G城

- 城（`kind: "castle"`）は未要塞化シールド 1 つに表向きで重なり要塞化する。シールド 0 または全要塞化済みなら実行不可。要塞化された城はシールドとして数えず、支えのシールドが離れると SBA で墓地へ。ブレイク時は S・トリガー処理後に墓地へ置かれる。
- G城（`special_types: ["galaxy"]`）は新しい表向きシールドとして置かれ、シールド 0 でも実行可。表向き G城がシールドを離れる時は置換でかわりに墓地へ（ブレイクイベントは発生しない）。裏向きになれば通常シールド扱い。
- G城は要塞化しないため `visible_shields()` に自分自身が盾として現れる。「自分の他のシールド」を対象にする効果では `exclude: {"ref": "source"}` で除外する。
- **城の誘発には `active_zones: ["shield"]` と `active_if`（`shield_face_up`）の両方が必要**。`active_zones` 未指定の誘発はバトルゾーン前提で一切発火しない。
- 城と G城は同じカードタイプ（城）なので「異なるカードタイプ」参照では同時に選べない。

## 進化・NEO

- 進化はカードを置き換えず `evolution_sources` に進化元を保持する。バトルゾーンの表向き一覧にはトップのみ。領域移動時は束ごと動き、移動後に `evolution_sources` は解除される。
- **クリーチャー選択とカード選択は別物**: `bounce_*_creature` は進化元を候補にせず束ごと戻す。`bounce_*_battle_card` は進化元も候補になり、進化元だけ・トップだけを戻せる。トップだけが離れた後は上から `can_exist_in_battle_alone()` を確認して再構成し、単独で存在できないカードは墓地へ。
- 進化元のパワー修正等の残存効果は進化先へ引き継ぎ、進化元だけが離れる時は戻す。
- NEO は召喚後に進化元を選ぶか確認され、選ばなければ通常クリーチャー（召喚酔いあり）。下のカードがなくなると `is_evolution` が `False` に戻り、同ターンに出ていれば召喚酔いが再適用される。
- ツインパクトが離れる時・進化元になる時は面選択をニュートラルに戻す。進化元のツインパクトはクリーチャー/呪文として扱わない。

## ツインパクト

- 「カード全体」と「使用中の面」を分離。マナゾーン等では両面の文明の結合（`get_all_civilizations()`）、使用時は面ごとのコスト・文明。
- `execute_card_from_hand` 等で `type: "element"` ならクリーチャー面、`"non_creature"` なら呪文面のコストを参照して実行する。

## クロスギア

- ジェネレート（単体で出す）とクロス（クリーチャーへ装備）はそれぞれ別にコストを支払う。再クロスで付け替え可（1 ギア = 同時に 1 体、1 体 = 複数ギア可）。
- クロスは独立オブジェクト（`crossed_to` 参照のみ）。クロス先が離れてもギアは場に残り未クロスへ。対象不適正の解除は SBA。
- クロス状態の変更は必ず `set_crossed_to()` 経由（`on_cross` / `on_uncross` で付与能力を適用・解除）。

## 能力の付与と無視

- **真の付与**（「与える」「得る」）は能力インスタンスを実際に `creature.abilities` へ追加する（`temporary_ability` 効果、`cross_grant`、常在型の `ScopedGrantAbility`）。`has_ability()` で参照され、能力無視の影響を**受ける**。
- **擬似付与**（「A が B した時、C する」）は付与元カードの能力として処理し、能力無視の影響を受けない。
- スコープ付与（`grant_ability` / `ScopedGrantAbility`）は SBA の `reconcile()` で対象の増減・付与元の状態変化に追従する（冪等）。付与する能力が置換効果型なら置換収集経由、常在型なら真の付与。
- 能力無視中は `has_ability()` が常に `False`。付与元が無視されると付与全体が無効、対象が無視されるとその対象への付与だけ無効。「攻撃できない」等の能力でない一時効果は無視の影響を受けない。

## ハイパーモード・ドリーム

- 解放は `unlock_hyper_mode_until_next_turn_start()`、次の自分のターン開始時に自動ロック。離れた時も `reset_battle_state()` で戻る。解放中の再解放は不可（期限も更新されない）。
- 失効判定は `hyper_mode_registered_turn`（解放時のターン番号）＋コントローラー一致で行う。`hyper_mode_expires_on_turn` は通常進行を仮定した目安値（表示・後方互換用）で、実判定には使わない。これにより追加ターン（[[turn-manager]]の挿入ターン）が挟まっても、コントローラーの最初のターン開始で正しく失効する。
- `z_rush` は誘発ではなく、シールドが離れた時に SBA が即時解放する（`ZoneChangeEvent` 発行前）。シールドが離れた後に出た Z ラッシュ持ちは過去の移動を参照しない。
- 「ハイパーモード中だけ」は `active_if: {"type": "source_has_state", "state": "hyper_mode"}`。
- 同名ドリーム・エレメントは自分のバトルゾーンに重複して出せない（進化元として無視されている場合を除く。ただし同名を進化元にして同名ドリーム進化は出せない）。
- 複数ブレイカーが同時に有効な場合、ブレイク直前に枚数を選択する（`ShieldBreakChoiceEvent` → `ChoiceManager`）。
- `cannot_cast_spells` は S・トリガー詠唱にも適用される（宣言候補化の前と解決直前に検証）。

## 期間付き効果

- `EnchantEffect`（カードへの一時付与）は、付与時のゾーンから移動した時・保留状態になった時に失われる。進化元になった場合は進化先へ転送され、退化で元へ戻る。
- `freeze_untap` はターン開始時の通常アンタップだけを止め、`lock_untap` は効果によるアンタップも止める。「それはアンタップしない」は `lock_untap`。
- 期間は `until_end_of_turn` / `until_end_of_opponent_turn` / `until_start_of_controller_turn` 等。`DurationEffectManager` がターン進行で自動クリーンアップする。

## キーワード個別の注意

- `escape` は `DestroyAttemptEvent` の置換。置換が成立すると `DestroyEvent` を発行しないため「破壊された時」は誘発しない。
- `ninja_strike` は宣言時にマナ条件を確認しない（解決時に確認、不足なら手札に残り使用済みにならない）。1 攻撃中に 1 回成功するまで複数宣言可。召喚したシノビはターン終了時に山札の下へ（その時点でシノビ種族を持たない場合を除く）。
- `g_zero` はコストを 0 にせず支払いだけをスキップする。通常の召喚として `SummonEvent` を発火する。
- `alternative_summon_cost` の支払い順: 追加コスト（クリーチャー破壊等）→ マナタップ → 召喚。`active_zone` と `requirements` を満たす時だけメインステップ候補に出る。
- `tap_to_play` は置換ではなく、置換チェック後・入場イベント発行前にタップ状態にする常在能力。`optional: true` で持ち主に確認できる。
- `shield_force` は選んだシールドをオブジェクト同一性 + `zone_change_counter` で追跡する。同じカードが再度シールドになっても効かない。
